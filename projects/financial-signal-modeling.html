Financial Signal Modeling

A research-style pipeline that treats market data and financial news as a coupled signal: sentiment, volatility, return behavior, cyclical structure, and explanation.

GitHub Profile

Problem

Financial news moves quickly, but most dashboards stop at surface-level sentiment. The goal of this project is to build a pipeline that connects news sentiment to short-horizon market behavior while preserving enough structure to explain what the model sees.

Method

The pipeline combines financial news sentiment with equity price data, engineers lagged sentiment and volatility features, and uses spectral analysis through periodograms to search for cyclical behavior. Autoregressive models are then used to evaluate temporal dependencies in returns and sentiment.

Interpretation Layer

Instead of stopping at model output, the project includes a fine-tuned 3B language model concept using LoRA / QLoRA to generate structured financial interpretations. The point is not just to forecast, but to create disciplined explanations of what signals might mean.

What This Shows

This project demonstrates time-series thinking, feature engineering, financial data literacy, and the ability to connect statistical modeling with human-readable interpretation.
