Real-time pairs trading.

A mean-reversion trading system built around cointegration, ADF tests, z-score thresholds, bootstrap uncertainty, and backtested entry/exit rules.

§ problem

When two prices want to come home together.

Pairs trading rests on a simple bet: find two assets whose price spread behaves like it wants to revert to a long-run mean, then trade the deviations from that mean. The mathematical object underneath is an Ornstein–Uhlenbeck-style process — a stochastic spread pulled back toward its equilibrium with some half-life. Everything in this project flows from that single picture.

View source on GitHub github.com/jortega4245/pairs-trading ↗
§ statistical frame

The plumbing.

  • Engle–Granger cointegration to identify pairs with a stable long-run linear relationship.
  • Augmented Dickey–Fuller (ADF) tests on the residual spread to evaluate stationarity.
  • Half-life estimation from an AR(1) fit on the spread — how fast does it revert?
  • z-score thresholds for entry, exit, and stop signals.
§ uncertainty

Bootstrap, because Fisher won't help here.

  • Parametric and non-parametric bootstrap resampling on backtest returns.
  • Confidence intervals for half-life and expected spread return.
  • Interval-based caution during high-volatility regimes to reduce false positives.
  • Walk-forward validation to avoid look-ahead leakage.
§ result

Backtested, not just hand-wavy.

The strategy is implemented in Python with optimized entry and exit rules, producing improved risk-adjusted performance — the resume version of the project reports a Sharpe ratio above 1.6. The interesting work here was not chasing return; it was being honest about how much of the Sharpe ratio is signal and how much is sample.

§ stack

Tooling.

Python, pandas, NumPy, statsmodels, ADF tests, Engle–Granger cointegration, bootstrap resampling, and a custom backtesting loop.
